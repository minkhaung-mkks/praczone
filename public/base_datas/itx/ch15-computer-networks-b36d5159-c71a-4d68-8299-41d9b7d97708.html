1. Introduction to Computer Networks

  • Computer Networks allow computers to communicate and share resources, enabling data exchange across long distances or localized areas.
    • Local Area Networks (LANs): Usually restricted to a small geographical area like an office building or home. LANs typically use Ethernet or Wi-Fi.
    • Wide Area Networks (WANs): Cover larger geographical areas, connecting devices over long distances, often through the public internet or leased telecommunications lines.
    • The Internet: A global network connecting millions of private, public, academic, business, and government networks.
Importance of Networking:
  • It allows resource sharing such as files, printers, and internet access.
  • Facilitates communication between people via email, instant messaging, and video conferencing.
  • Supports business processes, enabling online commerce, remote work, and global collaboration.

2. Types of Networks

  • Personal Area Network (PAN): A network used for communication among devices such as smartphones, laptops, and tablets within a short range (e.g., Bluetooth connections).
  • Metropolitan Area Network (MAN): Larger than a LAN but smaller than a WAN, typically spanning a city. Used by universities or companies with multiple offices in a metropolitan area.
Topologies in Networking:
  • Bus Topology: All computers share a single communication line.
    • Pros: Easy to implement, cost-effective.
    • Cons: If the central line fails, the whole network fails.
  • Ring Topology: Each computer is connected to two others, forming a circular path for data.
    • Pros: Data packets can travel in one or both directions.
    • Cons: A break in the ring can disrupt communication.
  • Star Topology: All computers are connected to a central hub.
    • Pros: Easy to manage and troubleshoot, failure of one device doesn’t affect the others.
    • Cons: The central hub represents a single point of failure.

3. The OSI Model and TCP/IP Protocol Stack

OSI Model:
  • The Open Systems Interconnection (OSI) model has seven layers, each serving a specific function in network communication:
    1. Physical Layer: Transmits raw bit streams over a physical medium (e.g., electrical signals, radio waves).
    2. Data Link Layer: Responsible for node-to-node data transfer and error detection/correction.
    3. Network Layer: Handles routing and forwarding of data.
    4. Transport Layer: Ensures reliable transmission of data segments between devices.
    5. Session Layer: Manages sessions between applications (e.g., maintaining connections in a video call).
    6. Presentation Layer: Translates data formats (e.g., encryption, decryption, data compression).
    7. Application Layer: Provides network services to end-users (e.g., email, file transfer, web browsing).
TCP/IP Model:
  • Similar to OSI but condensed into four layers:
    1. Link Layer: Combines OSI’s Physical and Data Link layers.
    2. Internet Layer: Corresponds to OSI’s Network layer.
    3. Transport Layer: Same as OSI’s Transport layer.
    4. Application Layer: Combines OSI’s Session, Presentation, and Application layers.

4. Physical Layer: Media and Transmission Methods

  • Copper Cables: Traditional Ethernet cables (e.g., Cat5, Cat6) transmit data as electrical signals. Copper cables are cheap and reliable for short distances (up to 100 meters for Cat6 cables).
  • Fiber Optic Cables: Use light to transmit data, allowing for much faster speeds and longer distances compared to copper cables. Typically used in WANs or backbone connections in large networks.
  • Wireless Transmission: Data is transmitted as radio waves (Wi-Fi) or microwaves (cellular networks). Wireless networks provide mobility but are subject to interference and security vulnerabilities.
Signal Types:
  • Analog Signals: Continuous signals, often used in older transmission methods.
  • Digital Signals: Discrete signals, often represented as binary (1s and 0s), and used in modern networking.

5. Data Link Layer: MAC Addressing and Error Detection

  • Media Access Control (MAC) Address: A unique identifier assigned to network interfaces for communication on a physical network. MAC addresses operate at the Data Link layer and help ensure that data reaches the correct device within a local network.
  • Error Detection:
    • Parity Bits: A simple form of error detection where an extra bit is added to data to make the number of 1s either even (even parity) or odd (odd parity).
    • Cyclic Redundancy Check (CRC): A more sophisticated error-detection algorithm that computes a checksum for a data packet and verifies that the data wasn’t corrupted during transmission.

6. Network Layer: IP Addressing and Routing

  • IP Addressing: Internet Protocol (IP) addresses uniquely identify devices on a network. IPv4 is the most widely used, though it is being replaced by IPv6 due to address exhaustion.
    • IPv4 (32-bit): Uses four sets of decimal numbers (e.g., 192.168.1.1).
    • IPv6 (128-bit): Uses hexadecimal notation to allow a vastly larger address space.
  • Routing: Routers determine the best path for data packets to travel from the source to the destination across multiple networks.
    • Static Routing: Manually configured routes, suitable for small or fixed networks.
    • Dynamic Routing: Routers communicate to discover the best paths dynamically. Routing protocols like RIP, OSPF, and BGP help routers share information about network topology.

7. Transport Layer: TCP and UDP

  • TCP (Transmission Control Protocol): A connection-oriented protocol that ensures data is reliably delivered to the recipient. It uses sequence numbers, acknowledgments, and retransmissions to guarantee data integrity.
  • UDP (User Datagram Protocol): A connectionless protocol that offers faster data transmission but without reliability guarantees. UDP is often used in real-time applications like video streaming or online gaming where speed is more critical than perfect accuracy.
Ports and Connections:
  • Ports identify specific processes or services within a computer. For example, web servers commonly use Port 80 for HTTP traffic and Port 443 for HTTPS traffic.

8. Application Layer Protocols

  • HTTP/HTTPS: Used for accessing and transmitting web pages. HTTPS adds a layer of encryption via SSL/TLS to ensure secure communication between the client and server.
  • SMTP (Simple Mail Transfer Protocol): Used for sending emails between mail servers.
  • DNS (Domain Name System): Resolves human-readable domain names (e.g., www.google.com) to IP addresses so that users can access websites without needing to memorize numeric addresses.

9. Networking Devices

  • Router: Directs traffic between different networks based on IP addresses.
  • Switch: Operates at the Data Link layer, forwarding data within a LAN based on MAC addresses.
  • Hub: A basic networking device that broadcasts data to all devices on the network, regardless of the destination.
  • Firewall: A security device that filters incoming and outgoing network traffic to protect a network from unauthorized access.

10. Network Security

  • Encryption: Converts data into a code to prevent unauthorized access during transmission. This ensures confidentiality and data integrity.
  • Firewalls: Can be software-based or hardware-based, and are essential for blocking unauthorized traffic and protecting networks from external attacks.
  • VPN (Virtual Private Network): Encrypts a user’s internet connection, allowing them to browse privately and securely, especially over public networks.

11. Wireless Networking

  • Wi-Fi (Wireless Fidelity): A technology that allows
devices to connect to the internet wirelessly using radio waves. Standards include 802.11n and 802.11ac, which offer faster speeds and greater range compared to earlier versions.
  • Bluetooth: Short-range wireless communication, typically used for connecting peripherals like keyboards, mice, and headphones.

12. Network Performance and Troubleshooting

  • Bandwidth: The maximum rate at which data can be transferred across a network. Measured in Mbps (Megabits per second).
  • Latency: The time it takes for a data packet to travel from the source to the destination. High latency can cause delays in real-time applications like gaming or video conferencing.
  • Packet Loss: Occurs when data packets fail to reach their destination, typically due to network congestion or faults in the transmission medium.
