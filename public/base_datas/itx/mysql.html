1. Introduction to SQL

Structured Query Language (SQL) is a standardized programming language used for managing relational databases and performing various operations on the data within them. SQL allows you to create, manipulate, and retrieve data stored in relational databases.

Key Concepts:

  • Data Definition Language (DDL): Commands that define the database structure.
  • Data Manipulation Language (DML): Commands that manipulate data within tables.
  • Data Control Language (DCL): Commands that control access to data.
  • Transaction Control Language (TCL): Commands that manage transactions in the database.

2. Understanding Relational Databases

A relational database stores data in tables composed of rows and columns. It uses keys to establish relationships between tables.

Key Components:

  • Table: A collection of related data entries.
  • Row (Record): A single, data item in a table.
  • Column (Field): A set of data values of a particular type.
  • Primary Key: Uniquely identifies each record.
  • Foreign Key: Links records between tables.

3. Getting Started with MySQL on Windows

System Requirements

  • Operating System: Windows 7 or later
  • RAM: Minimum 512 MB (1 GB recommended)
  • Disk Space: Approximately 500 MB for installation

Downloading and Installing MySQL

  1. Download MySQL Installer:
    • Visit the MySQL Downloads page.
    • Choose "MySQL Installer for Windows" (preferably the full version).
  2. Run the Installer:
    • Execute the downloaded .msi file.
  3. Choose Setup Type:
    • Select "Developer Default" to install MySQL Server, MySQL Workbench, and other useful tools.
  4. Check Requirements:
    • The installer will check for prerequisites like Visual C++ Redistributable. Install them if prompted.
  5. Installation:
    • Click "Execute" to install the selected products.

Configuring MySQL Server

  1. Type and Networking:
    • Choose "Standalone MySQL Server".
    • Default port is 3306.
  2. Authentication Method:
    • Use "Use Strong Password Encryption".
  3. Accounts and Roles:
    • Set a root password.
    • Optionally, create additional users.
  4. Windows Service:
    • Configure MySQL as a Windows Service for automatic startup.
  5. Apply Configuration:
    • Click "Execute" to apply the configuration settings.

Using MySQL Workbench

MySQL Workbench is a graphical tool for working with MySQL databases.
  1. Launching MySQL Workbench:
    • Open MySQL Workbench from the Start menu.
  2. Connecting to MySQL Server:
    • Click on the local instance MySQL80 (or the version you installed).
    • Enter the root password you set during installation.
  3. Navigating the Interface:
    • Navigator Pane: Shows database schemas and objects.
    • Query Editor: Area to write and execute SQL queries.

4. Fundamentals of SQL

SQL Syntax and Conventions

  • Case Sensitivity: SQL keywords are not case-sensitive (SELECT is the same as select).
  • Statements End with Semicolons: ; indicates the end of a statement.
  • Comments:
    • Single-line: -- comment
    • Multi-line: /* comment */

Data Types in MySQL

  • Numeric Types: INT, BIGINT, FLOAT, DOUBLE, DECIMAL
  • String Types: CHAR(n), VARCHAR(n), TEXT, BLOB
  • Date and Time Types: DATE, TIME, DATETIME, TIMESTAMP
  • Boolean Type: BOOLEAN (alias for TINYINT(1))

5. Data Definition Language (DDL)

DDL commands define the structure of the database, including creating, altering, and deleting databases and tables.

Creating Databases

CREATE DATABASE database_name;
Example:
CREATE DATABASE school_db;

Creating Tables

CREATE TABLE table_name (
  column1 datatype [constraints],
  column2 datatype [constraints],
  ...
);
Example:
CREATE TABLE students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  first_name VARCHAR(50) NOT NULL,
  last_name VARCHAR(50) NOT NULL,
  birth_date DATE,
  email VARCHAR(100) UNIQUE
);

Altering Tables

  • Add a Column:
    ALTER TABLE table_name
ADD COLUMN column_name datatype [constraints];
  • Modify a Column:
    ALTER TABLE table_name
MODIFY COLUMN column_name new_datatype;
  • Drop a Column:
    ALTER TABLE table_name
DROP COLUMN column_name;

Dropping Tables and Databases

  • Drop Table:
    DROP TABLE table_name;
  • Drop Database:
    DROP DATABASE database_name;

6. Data Manipulation Language (DML)

DML commands are used to manipulate data within tables.

Inserting Data

INSERT INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...);
Example:
INSERT INTO students (first_name, last_name, birth_date, email)
VALUES ('John', 'Doe', '2000-05-15', 'john.doe@example.com');

Updating Data

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;
Example:
UPDATE students
SET email = 'john.d.new@example.com'
WHERE student_id = 1;

Deleting Data

DELETE FROM table_name
WHERE condition;
Example:
DELETE FROM students
WHERE student_id = 1;

Selecting Data

SELECT column1, column2, ...
FROM table_name
WHERE condition;
Example:
SELECT first_name, last_name
FROM students
WHERE birth_date > '1999-12-31';

7. Advanced Querying Techniques

Filtering with WHERE Clause

Use the WHERE clause to filter records based on conditions.
Example:
SELECT * FROM students
WHERE birth_date BETWEEN '2000-01-01' AND '2002-12-31';

Sorting Results with ORDER BY

Order the result set using ORDER BY.
Example:
SELECT first_name, last_name
FROM students
ORDER BY last_name ASC, first_name DESC;

Limiting Results with LIMIT

Limit the number of records returned.
Example:
SELECT * FROM students
LIMIT 5;

Using Aliases

Rename columns or tables for readability.
Example:
SELECT s.first_name AS 'First Name', s.last_name AS 'Last Name'
FROM students AS s;

Joining Tables

Combine rows from two or more tables.
Example:
SELECT students.first_name, courses.course_name
FROM students
INNER JOIN enrollments ON students.student_id = enrollments.student_id
INNER JOIN courses ON enrollments.course_id = courses.course_id;

Grouping Data with GROUP BY

Group records to perform aggregate calculations.
Example:
SELECT course_id, COUNT(*) AS num_students
FROM enrollments
GROUP BY course_id;

Having Clause

Filter groups based on conditions.
Example:
SELECT course_id, COUNT(*) AS num_students
FROM enrollments
GROUP BY course_id
HAVING COUNT(*) > 5;

8. Functions and Expressions

Functions perform calculations on data.

Aggregate Functions

  • COUNT(): Returns the number of rows.
  • SUM(): Returns the total sum.
  • AVG(): Returns the average value.
  • MIN() / MAX(): Returns the minimum or maximum value.
Example:
SELECT COUNT(*) FROM students;

String Functions

  • CONCAT(): Concatenates strings.
  • UPPER() / LOWER(): Converts to uppercase or lowercase.
  • SUBSTRING(): Extracts a substring.
Example:
SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM students;

Date and Time Functions

  • CURDATE(): Returns current date.
  • DATEDIFF(): Calculates difference between dates.
Example:
SELECT DATEDIFF(CURDATE(), birth_date) AS days_alive FROM students;

Mathematical Functions

  • ROUND(): Rounds a number.
  • FLOOR() / CEILING(): Rounds down or up to the nearest integer.
Example:
SELECT ROUND(score, 2) FROM exams;

9. Subqueries and Derived Tables

A subquery is a query nested inside another query.
Example of Subquery:
SELECT first_name, last_name
FROM students
WHERE student_id IN (SELECT student_id FROM enrollments WHERE course_id = 1);

10. Views and Indexes

Creating and Managing Views

A view is a virtual table based on the result set of an SQL statement.
Creating a View:
CREATE VIEW view_name AS
SELECT columns
FROM table_name
WHERE condition;
Example:
CREATE VIEW student_emails AS
SELECT first_name, last_name, email
FROM students
WHERE email IS NOT NULL;

Creating and Managing Indexes

An index improves the speed of data retrieval.
Creating an Index:
CREATE INDEX index_name ON table_name (column1, column2, ...);
Example:
CREATE INDEX idx_last_name ON students (last_name);

11. Stored Procedures and Functions

Creating Stored Procedures

A stored procedure is a set of SQL statements that can be saved and reused.
Example:
DELIMITER //

CREATE PROCEDURE GetStudentEmail (IN sid INT, OUT email VARCHAR(100))
BEGIN
  SELECT s.email INTO email
  FROM students s
  WHERE s.student_id = sid;
END //

DELIMITER ;

Creating User-Defined Functions

A function returns a value and can be used in SQL statements.
Example:
DELIMITER //

CREATE FUNCTION GetAge (birth DATE)
RETURNS INT
BEGIN
  DECLARE age INT;
  SET age = TIMESTAMPDIFF(YEAR, birth, CURDATE());
  RETURN age;
END //

DELIMITER ;

12. Transactions and Concurrency Control

Transactions ensure data integrity.
Commands:
  • START TRANSACTION
  • COMMIT
  • ROLLBACK
Example:
START TRANSACTION;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

COMMIT;

13. User Management and Security

Creating Users

CREATE USER 'username'@'hostname' IDENTIFIED BY 'password';
Example:
CREATE USER 'dionne'@'localhost' IDENTIFIED BY 'password123';

Granting Privileges

GRANT privileges ON database.table TO 'username'@'hostname';
Example:
GRANT SELECT, INSERT ON school_db.* TO 'dionne'@'localhost';

Restore a Database

mysql -u username -p database_name < backup_file.sql
